January 3rd 2005

Jrobots is a clone of a DOS game named Crobots. Some robots fight in an arena, firing missiles and avoiding enemies' projectiles. At the end only one will survive!
This game is involving, but it's not interactive: you must develop the AI algorithms of your own robot using the Java language, then you upload it in the arena and follow on-line its fights against other robots.

Does this game really work?
It's simpler to see than believe, so visit the page of the Battle Applet.

How can you play?
It's simpler to do than explain, so download the Jrobot SDK or read the on-line documentation to get more information.

You'll definitely love this game! Why?
For an answer read the Comments of a Veteran.

Could this game be improved?
Check the current extensions to the original API and propose your ideas to the author of Jrobots or to the mailing list.
Jrobots is not a static game. Improvements were proposed and the game acquires value in time. For some examples read the proposal of the current match selection mechanism and the notes about the new virtual clock generator.
